Solo Project Web Game 2D Arcade

Wall Jump Frenzy

Timeframe: 2 Weeks
Role: Game Developer and Designer
Tools: Visual Studio Code, PixiJS, Howler.JS
Skills: Javascript, HTML, CSS

A simple arcade-clicker game designed for the web with Javascript, PixiJS, and Howler.JS. The goal is to collect as many coins as possible while avoiding all of the obstacles that spawn in while you play. To play, just left-click anywhere on the game screen to jump from wall to wall. The block's jumping movement is physics based and it will also rotate in the air, always landing flush with the wall. Obstacles will spawn in with their hitboxes deactivated so that the player can plan their movement around the obstacles. After a short period of time each obstacle will then activate and move across the screen. Collision with an active obstacle will end the game unless the player has picked up a shield, which increases the player's health by one. After the player loses its shield, it will gain invincibility for a short period.

Perfecting the block's movement as much as I could was the main challenge of this game and I went through two different iterations of jumping mechanics and rotation mechanics. At first for jumping, I started with the physics kinematics equation for delta position using an initial velocty and an acceleration, which resulted in an arch, but did not allow for jumping multiple times in the air all that easily. So I switched to a system where acceleration and velocty were calculated separately each frame so that each jump, velocity could be changed so the block restarts its arch. For rotation I initially started rotating the block constantly until it hits a wall and then it would rotate quickly to be flush with that wall. This method didn't look very nice and sometimes took longer than it should have to jump to the right rotation. I modified the code to instead do 1 full rotation and stop. This still kept the look that I wanted while completely removing all the rotation issues I was experiencing.

    Challenges:
  • Perfecting the block's jump speed and force to feel fun, balanced, and not require too many clicks.
  • Implementing a rotation during each jump that adds to the visual flair but doesn't interrupt the gameplay by having different rotations when landing on the walls.
  • Setting up all the obstacles in the game to be randomly placed, to have a random speed, and a random duration that are all within set parameters.

    What I did well:
  • The game starts out easier with less obstacles to dodge and gets harder as the player collects more coins. Multiple tiers of difficulty are incorperated which come into effect after certain coin requirements.
  • The game is endless and allows for very high high-scores. It's fun to challenge other people and see who is able to get the highest score.
  • The rules are very simple to understand and can be understood through only playing the game.

    What I would change/improve:
  • Increase the block's jump height while keeping the falling the velocity around the same.
  • Give the block a 360 degree rotation everytime it jumps, and increase the speed of the rotation if it's too close to the wall.
  • Add more powerups for the player in addition to the shield.
  • Add more obstacles with different movement patterns.
